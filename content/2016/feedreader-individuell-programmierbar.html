Feedreader: individuell programmierbar

Ich lese gerade einen Artikel über Calliope, einen der Computer der unser Schulsystem revolutionieren kann. Wichtige Aussage in einem verlinkten Video: Code ist ein gesellschaftlicher Faktor.

Mir würde es gefallen, wenn es einen Feed-Reader unter der vollen Kontrolle des Users gäbe. Ein Feedreader, der nicht nur offene Web-Feeds (wie RSS und Atom) beherrscht, sondern auch Facebook, Twitter und andere Third-Party-Silos. Allein das wäre schon schon toll.

Dabei sollte es jedoch nicht bleiben. Ein weiteres wichtiges Feature, wären programmierbare Filter. Du kannst an einem Beitrag Einstellungen vornehmen: Beiträge wie diesen verbergen. oder Beiträge wie diesen häufiger anzeigen. Das System soll dann jedoch nicht selbst entscheiden, weshalb diese Beiträge nicht mehr angezeigt werden sollen. Stattdessen soll der User entscheiden, wie der Filter aussehen soll.

Zum Beispiel will ich vielleicht nicht lesen, was mein Kollege bei Twitter schreibt, sein Blog interessiert mich jedoch. Hier soll nicht diese Person komplett ausgeblendet werden, sondern diese Person in einem Medium. Vielleicht interessiert mich jedoch nur nicht, welche Strickmuster mein Kollege bei Twitter teilt und ich blende nur den passenden Hashtag aus.

Diese Einstellungen sollten für den Einstieg einfach per Mausklick konfigurierbar sein. Es soll jedoch Code in einer leichten Skriptsprache (bspw. Lua) erzeugt werden. Dieser Code kann später direkt bearbeitet und erweitert werden.

Auf diese Weise kann ich als Benutzer lernen, wie diese Algorithmen arbeiten. Denkbar sind auch gewisse Standard-Algorithmen. Zum Beispiel mit einer integrierten Bookmark-Funktion. Ein Standard-Algorithmus könnte so funktionieren, dass Beiträge als wichtiger gewertet werden, die oft gespeicherten ähneln.
